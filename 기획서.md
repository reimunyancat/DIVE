.# DIVE 프로젝트 기획서

## 1. 프로젝트 개요

- **서비스명:** DIVE
- **부제:** Deep-dive Into Your Every Theme
- **슬로건:** 당신의 취향이 곧 목적지가 되는 여행

### 기획 배경:

- 기존 여행 서비스는 대중적인 관광 코스 위주의 추천에 집중되어 있어 애니메이션, 영화, 음식, 서브컬처 등 개인의 취향 중심 여행 니즈를 충분히 충족하지 못하고 있음.
- 또한 AI 기반 추천 서비스는 환각 문제로 정보 신뢰도에 한계를 가짐.

### 목표:

- 취향 기반 테마 여행 일정 자동 생성
- 다중 출처 검증을 통한 신뢰성 확보
- 지도 기반 직관적 일정 관리 및 공유 제공

## 2. 서비스 개요

### 서비스 정의:

- DIVE는 사용자의 취향 키워드를 분석하여 검증된 테마 장소와 최적의 여행 일정을 자동으로 생성해주는 AI 여행 큐레이션 웹 서비스이다.

### 주요 대상:

- 애니/영화/드라마 성지순례 유저
- 음식·카페·전시·서브컬처 테마 여행자
- 여행 계획에 시간을 쓰기 어려운 개인 및 소규모 그룹

## 3. 주요 기능

- **3-1. 테마 분석 및 장소 추출:** 자유 입력 기반 테마 분석 후 관련 장소 후보 자동 도출
- **3-2. 다중 출처 팩트체크:** 위키, 공식 사이트, 블로그 등 다중 소스 검증 및 신뢰도 점수 제공
- **3-3. AI 일정 자동 생성:** 이동 거리, 체류 시간, 운영시간 고려 일정 설계
- **3-4. 인터랙티브 지도:** `Leaflet` 기반 마커, 경로 시각화 및 드래그 편집
- **3-5. 일정 저장 및 공유:** DB 저장 및 고유 URL 공유
- **3-6. (확장) 이미지 기반 장소 제안:** 사진 업로드 시 유사 장소 후보 제안

## 4. 서비스 흐름

1. 사용자 테마 입력
2. AI 테마 분석 및 장소 후보 도출
3. 자동 팩트체크 및 신뢰도 산정
4. 최적 일정 생성
5. 지도 시각화 및 편집
6. 저장 및 공유

## 5. 기술 스택

- **Frontend:** `Next.js`, `Tailwind CSS`, `Shadcn/UI`
- **Backend:** `Node.js` (`TypeScript`)
- **AI/LLM:** Google `Gemini`
- **Database:** `Supabase` (`PostgreSQL`, `pgvector`)
- **Maps:** `Leaflet.js`, `OpenStreetMap`
- **Routing:** `OSRM` / `GraphHopper`
- **Deployment:** `Vercel`

### 5.1. 기술 스택 선택 이유

#### **DB & Backend: Supabase**

저희가 Supabase를 선택한 이유는 명확합니다. **Firebase의 개발 편의성과 PostgreSQL의 강력한 성능 및 확장성**을 모두 가져가고 싶었기 때문입니다.

Supabase는 단순한 BaaS가 아닙니다. 내부에 검증된 PostgreSQL을 품고 있어, DIVE의 핵심인 장소, 좌표, 일정 같은 복잡한 관계형 데이터를 안정적으로 다룰 수 있습니다. 특히 `pgvector` 확장 지원은 ‘케이온 성지순례’ 같은 테마 기반의 AI 검색 기능을 구현할 핵심 열쇠가 될 겁니다.

인증, 스토리지, API 자동 생성 같은 부가 기능들은 백엔드 개발 리소스를 획기적으로 줄여주어, 저희 팀이 서비스의 핵심 가치에 더 집중할 수 있게 해줍니다. 만약 Supabase를 사용하지 않는다면 이 모든 것을 직접 구현해야 하고, 이는 MVP 개발 속도를 심각하게 저해했을 것입니다.

#### **지도: Leaflet.js + OpenStreetMap**

지도 기능은 DIVE의 핵심 경험입니다. 저희는 **비용 문제와 커스터마이징의 한계** 때문에 구글맵 대신 Leaflet.js와 OpenStreetMap(OSM) 조합을 선택했습니다.

구글맵은 일정 트래픽을 넘어서면 바로 과금이 시작되어 MVP 단계에선 부담이 큽니다. 반면 OSM의 풍부한 무료 지도 데이터를 경량 라이브러리인 Leaflet.js로 자유롭게 가공하면, ‘캐릭터 모양 마커’처럼 DIVE만의 독창적인 지도 UX를 만들 수 있습니다. AI가 생성한 경로를 지도 위에 즉시 시각화하는 등, 서비스의 상상력을 제한 없이 펼칠 수 있다는 점이 가장 큰 매력입니다.

#### **경로 계산: OSRM / GraphHopper**

AI가 추천한 장소들을 연결해 최적의 동선을 짜려면, 실제 이동 시간을 계산할 경로 엔진이 필수입니다. 저희는 오픈소스인 OSRM이나 GraphHopper를 자체 서버에 구축하여 이 문제를 해결할 계획입니다. 이를 통해 **구글 길찾기 API 같은 유료 서비스에 의존하지 않고도,** 도보나 차량 이동을 고려한 현실적인 소요 시간을 계산하여 일정의 완성도를 높일 수 있습니다.

#### **전체 스택 요약**

|     | 구분           | 스택                     | 선택 이유                                                |
| --- | -------------- | ------------------------ | -------------------------------------------------------- |
|     | **Frontend**   | Next.js, Tailwind/Shadcn | SEO, 성능, 빠른 UI 개발을 위한 검증된 조합               |
|     | **Backend**    | Node.js (TypeScript)     | LLM 연동 및 비동기 API 처리에 가장 유연하고 강력함       |
|     | **AI**         | Google Gemini            | 멀티모달 입력과 빠른 응답 속도로 서비스의 핵심을 담당    |
|     | **Database**   | Supabase (PostgreSQL)    | DB, Auth, API를 한 번에 해결해주는 가장 효율적인 선택    |
|     | **Map**        | Leaflet.js + OSM         | 무료 사용과 자유로운 커스터마이징으로 DIVE만의 지도 구현 |
|     | **Routing**    | OSRM / GraphHopper       | 자체 서버 구축으로 비용 없이 현실적인 경로 계산          |
|     | **Deployment** | Vercel                   | Next.js 프로젝트를 가장 빠르고 안정적으로 배포           |
